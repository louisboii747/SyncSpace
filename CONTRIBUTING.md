# Contributing to SyncSpace

Thank you for your interest in contributing to SyncSpace.

## Getting Started

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make your changes.
4. Test your changes where possible.
5. Open a pull request with a clear description of the changes.

## Project Structure

```text
backend/   - Go synchronization backend
windows/   - Windows application
android/   - Android application
ios/       - iOS application
macos/     - macOS application
shared/    - Shared resources and documentation
docs/      - Project documentation
```

## Coding Guidelines

- Keep code simple and maintainable.
- Follow the existing project structure.
- Use descriptive commit messages.
- Document significant architectural decisions.
- Prioritize functionality and reliability before visual polish.

## Pull Requests

When submitting a pull request:

- Explain what was changed.
- Explain why the change was made.
- Include screenshots for UI changes where applicable.
- Keep pull requests focused on a single feature or fix.

## Issues

Bug reports and feature requests are welcome through GitHub Issues.

Before opening a new issue, please check whether a similar issue already exists.
